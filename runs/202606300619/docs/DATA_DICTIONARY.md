Select/Omit: Likely selects subset of patients (e.g., facility 5H).\n\n### HAPIRNK — Patient Ranking Logical\n\n- Based on PF: OAPIRNK or equivalent.\n- Format Source: OAPIRNKR\n- Key Fields: PATID, RANK\n- Select/Omit: May omit voided or inactive rankings.\n\n### HXPBNFIT — Benefit Logical\n\n- Based on PF: OXPBNFIT\n- Format Source: OXPBNFITR\n- Key Fields: PATID, BNFTID\n- Select/Omit: May select only active benefits.\n\n### HXPNSTN — Institution Logical\n\n- Based on PF: OXPNSTN\n- Format Source: OXPNSTNR\n- Key Fields: INSTID\n- Select/Omit: May select only active institutions.\n\n### HXLTABLD — Table Logical\n\n- Based on PF: HXPTABLD\n- Format Source: HXPTABLDR\n- Key Fields: TABLEID, KEY\n- Select/Omit: General table view.\n\n### HXLTABLP — Table Parameter Logical\n\n- Based on PF: HXPTABLD\n- Format Source: HXPTABLDR\n- Key Fields: TABLEID, PARAM\n- Select/Omit: Parameterized view.\n\n### HXLTABLS — Status Logical\n\n- Based on PF: HXPTABLD\n- Format Source: HXPTABLDR\n- Key Fields: STATUS\n- Select/Omit: Status code view used for CABEQ literals.\n
