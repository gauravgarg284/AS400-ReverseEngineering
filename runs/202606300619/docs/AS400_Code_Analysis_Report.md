Observations\n\n- The system centers on HABADTE as the main orchestration program for patient management, ranking, benefits, and reporting.\n- Utility programs (XFXCNTR, XFXCYMD, XFXLDSC, XFXTABL, XFXGETID, XFXLEAP, XFXMRNROL) encapsulate reusable cross-cutting concerns such as counter handling, date validation, table lookups, and MRN rollover.\n- Data is distributed across PF/LF pairs with external text exports, suggesting batch-oriented feeds and reporting pipelines.\n- Table-driven logic via HXPTABLD and its LF variants enables flexible status codes, preferences, and control tables, which should map to configuration tables in SQL Server.\n- The presence of org-level PFs (HXPLVL1–HXPLVL6) and preference copy members (HXXAPPPRFP, HXXLEVEL) indicates a multi-level organization hierarchy with preference overrides per level.\n- PHI concentration in a small set of core files makes it possible to isolate and protect sensitive data in the modernized architecture.\n- No critical or high-risk programs are flagged by complexity_scores, but migration must still respect subtle indicator-based branching captured in business rules BR-001 through BR-019.\n
